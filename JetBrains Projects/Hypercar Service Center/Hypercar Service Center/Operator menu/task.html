Description

The system you're creating is almost ready for customers to use, but you also need to add features designed for the Hypercar operators who process the orders. One of their requirements is to be able to view the current length of each queue. For example, if the queue lengths are 3, 4, and 1, the following should be displayed:

  • Change oil queue: 3
  • Inflate tires queue: 4
  • Get diagnostic queue: 1

They also want the ability to select when to process the next customer. A simple button that can be clicked to complete this action will work just fine.

Objectives

You need to create an operator menu page containing three <div> elements with the content shown in the above list. The last element on the page should be a <form> with the method attribute equal to "post".

The form should contain a <button> element with the type attribute equal to "submit", and the button's label should be Process next:

<div>Change oil queue: 3</div>
<div>Inflate tires queue: 4</div>
<div>Get diagnostic queue: 1</div>

<form method="post">{% csrf_token %}
  <button type="submit">Process next</button>
</form>

The operator menu page should be available at <base_url>/processing, where <base_url> is the host.

Note that the processing URL doesn't contain a slash at the end, so your URL pattern shouldn't either. You can redirect GET requests to the right page with the help of RedirectView if you need to:

from django.views.generic import RedirectView

urlpatterns = [
   # some other patterns
   path('processing/', RedirectView.as_view(url='/processing'))
